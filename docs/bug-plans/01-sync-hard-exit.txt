BUG: `alter sync` aborts on skipped files — discards successfully parsed schema
SEVERITY: High
STATUS: Open

REPRODUCTION:
  # Given a project with 10 model files, one of which has a syntax error:
  $ alter sync
  # Output:
  #   Parsing sqlmodel models in app/…
  #   ⚠  Skipped: app/models/broken.py
  # Exit code: 1
  # schema.alter is NOT updated — the 9 good files are thrown away.

EXPECTED:
  Schema should be updated with the 9 successfully parsed files. The skipped
  file should be a warning, not a fatal error. Exit code 1 only if zero
  tables were parsed.

ROOT CAUSE:
  cli.py lines 288-291:
    if result.skipped_files:
        for fp in result.skipped_files:
            click.echo(f"  ⚠  Skipped: {fp}", err=True)
        sys.exit(1)

  This runs BEFORE the canvas-position preservation (line 294) and save
  (line 298). The parsed `new_schema` is discarded entirely.

  Compare with `alter init` (line 218-227) which warns about skipped files
  but continues to save — inconsistent behaviour between the two commands.

FIX PLAN:
  1. Remove the `sys.exit(1)` after the skipped-files warning loop.
  2. Let the function continue to preserve positions and save.
  3. After saving, if there were skipped files, exit with code 2 (partial
     success) so CI can still detect issues, but the schema is not lost.
  4. Print a summary: "Synced N tables (M files skipped with errors)."

  Diff sketch:
    if result.skipped_files:
        for fp in result.skipped_files:
            click.echo(f"  ⚠  Skipped (parse error): {fp}", err=True)
-       sys.exit(1)

    # Preserve canvas positions
    for tbl in new_schema.tables:
        ...
    new_schema.save(path)
    click.echo(f"  Synced {len(new_schema.tables)} tables → {path.name}")

+   if result.skipped_files:
+       sys.exit(2)

VERIFICATION:
  - Create a project with 2 model files, one valid and one with a SyntaxError.
  - Run `alter sync`.
  - schema.alter should contain the table from the valid file.
  - Exit code should be 2 (not 1).
  - Run `alter sync` with all-valid files → exit code 0.

TESTS:

  - Test exit code 2 vs 0.
